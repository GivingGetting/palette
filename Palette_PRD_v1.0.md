# Palette — 产品需求文档（PRD）

**版本** v1.1 · **日期** 2026-03-20 · **状态** 草稿

---

## 一、产品概述

### 1.1 背景

设计师在参考竞品或优秀案例时，往往需要手动记录色彩、字体、间距等风格要素，过程繁琐且难以系统化复用。大模型图像生成能力的成熟，使"以风格为输入"的生图工作流成为可能，但目前缺乏一个将风格提取与生图对比整合的专用工具。

### 1.2 产品定位

> 为设计师打造的 AI 风格解析引擎 —— 分析任意来源，提炼可复用的视觉语言。

**目标用户**：UI/UX 设计师、产品设计师、设计系统负责人、前端开发者

**核心价值主张**：
- 输入一个 URL 或截图 → 输出结构化风格报告（Style DNA）
- 将风格报告作为 prompt 注入，驱动多模型生图并排对比
- 所有解析结果自动归档，构建个人风格参考库

---

## 二、用户目标与痛点

| 痛点 | 当前解法 | Palette 解法 |
|------|----------|-------------|
| 手动整理竞品色彩/字体耗时 | Figma 手动取色 | 一键解析，18 项维度自动输出 |
| 风格参考散落在截图/书签里 | Notion 手动归档 | 解析后自动入库，结构化存储 |
| 不同 AI 模型生图效果难横向比较 | 逐个测试复制粘贴 | 同一 prompt 多模型并发，结果并排 |
| 无法将"风格"精确传达给 AI | 用文字描述风格，模糊 | Style DNA 直接注入 prompt |

## 三、功能模块

### 3.1 来源解析（核心功能）

**解析引擎架构：** 自研方案
- Playwright headless 浏览器抓取页面截图 + 完整 DOM/CSS 样式树
- 提取结果交由 Claude 进行语义分析，输出结构化 Style DNA JSON
- 图片上传路径：跳过抓取阶段，直接送入视觉分析模型

**输入方式：**
- URL 输入（自动抓取页面截图 + DOM 样式）
- 图片上传（支持拖拽，PNG / JPG / WebP）

**解析引擎输出 —— Style DNA 报告（18 项维度）：**

| 维度分类 | 具体项 |
|---------|--------|
| 色彩系统 | 主色、辅助色、背景色、强调色、语义色（成功/警告/错误） |
| 字体规范 | 主字体族、标题字重、正文字号/行高/字间距 |
| 空间系统 | 基础间距单位、常用间距值、容器最大宽度 |
| 圆角语言 | 按钮/卡片/输入框圆角值，是否偏好 pill 形态 |
| 组件语言 | 按钮形态、标签风格、表单风格、导航模式 |
| 整体风格 | 明暗模式偏好、信息密度、动效倾向（关键词标注） |

**状态设计：**
- 解析中：进度条 + 阶段说明（"正在抓取页面…" / "正在分析色彩…"）
- 解析完成：展示完整 Style DNA 卡片
- 解析失败：错误原因 + 重试入口

---

### 3.2 模型对比（差异化功能）

**流程：**
1. 用户输入文字描述（自然语言 prompt）
2. 可选：挂载一个 Style DNA 作为风格约束
3. 选择参与对比的模型（最多 4 个并发）
4. 点击"全部生成" → 各模型结果并排展示

**支持模型（v1.0）：**
- GPT-4o（OpenAI）
- Claude 3.5 Sonnet（Anthropic）
- Gemini 1.5 Pro（Google）
- Ideogram v2（备选）

**API Key 策略：** 用户自带（BYOK）
- 用户在设置页填入各模型的 API Key，平台加密存储
- 平台不承担生图 API 费用，用户按各模型官方定价自行消耗
- 未填入某模型 Key 时，该模型在对比页置灰不可选

**结果卡片包含：**
- 生成图预览
- 耗时
- 收藏 / 导出 / 重新生成操作

**Style DNA 注入规则：**
Style DNA 挂载后，系统自动将色彩、字体、圆角等关键参数拼入系统 prompt，用户无需手动描述风格细节。

---

### 3.3 资源库（留存功能）

- 每次解析自动归档，支持手动命名与打标签
- 卡片展示：来源缩略图、风格色盘、描述摘要、解析时间
- 支持筛选（按来源类型、添加时间、风格关键词）
- 支持排序（最近添加、最常调用）
- 单条记录可展开查看完整 Style DNA 报告
- 支持导出为 JSON / Figma Token 格式（v1.1 规划）

---

### 3.4 账户与记录

- 登录方式：邮箱 + Google OAuth
- **数据隔离：资源库、对比记录默认私有**；用户可手动对单条记录生成分享链接（只读），不做 SEO 公开索引
- **配额管理（按月重置）：**

| 套餐 | 解析次数 | 生图次数 | API Key | 价格 |
|------|---------|---------|---------|------|
| 免费版 | 20 次 / 月 | 50 次 / 月 | 用户自带 | 免费 |
| Pro 版 | 无限制 | 无限制 | 用户自带 | 待定 |

- 配额每月 1 日 00:00 UTC 重置，不累计到下月

## 五、核心用户流程

### 流程 A：首次解析

```
首页输入框 → 粘贴 URL → 点击"解析"
→ 解析进度页（约 8-15s）
→ Style DNA 报告页（完整 18 项）
→ 自动归档至资源库
→ 引导进入"模型对比"
```

### 流程 B：模型对比生图

```
资源库选择一条 Style DNA
→ 进入对比页，Style DNA 自动挂载
→ 输入场景描述
→ 选择 2-4 个模型
→ 点击"全部生成"
→ 并排查看结果
→ 收藏满意结果
```

### 流程 C：快速调取复用

```
资源库搜索 "stripe"
→ 打开 Style DNA 报告
→ 复制色彩 Token / 字体规范
→ 或直接挂载到新的生图任务
```

## 六、非功能性需求

| 项目 | 指标 |
|------|------|
| 解析响应时间 | P90 ≤ 15s（URL 输入） |
| 生图等待时间 | 各模型各自计时，超 30s 显示超时提示 |
| 页面首屏加载 | LCP ≤ 2.5s |
| 支持分辨率 | 1280px 以上桌面端（v1.0），移动端适配 v1.1 |
| 数据安全 | 用户上传图片仅用于解析，不用于模型训练 |
| 可用性 | SLA 99.5% |
