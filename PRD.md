# Palette — 产品需求文档（PRD）

**版本** v1.2 · **日期** 2026-03-22 · **状态** 进行中

---

## 一、产品概述

### 1.1 定位

> 为设计师打造的 AI 风格解析引擎 —— 分析任意来源，提炼可复用的视觉语言，驱动多模型生图对比。

**目标用户**：UI/UX 设计师、产品设计师、印刷品创作者、前端开发者

**核心价值**：
1. 输入任意网站 URL → Claude 提取结构化「Style DNA」（色彩、字体、版式、美学）
2. Style DNA 注入 Prompt → 多模型并发生图，结果并排对比
3. 所有解析结果自动归档，构建个人风格参考库
4. ComfyUI 本地生成支持高分辨率输出，满足 Printify 等印刷平台要求

---

## 二、用户痛点与解法

| 痛点 | 当前解法 | Palette 解法 |
|------|----------|-------------|
| 手动整理竞品色彩/字体耗时 | Figma 手动取色 | 一键解析，18 项维度自动输出 |
| 风格参考散落截图/书签 | Notion 手动归档 | 解析后自动入库，结构化存储 |
| 不同 AI 模型生图效果难横向比较 | 逐个测试复制粘贴 | 同一 prompt 多模型并发，结果并排 |
| 无法将"风格"精确传达给 AI | 用文字描述，模糊 | Style DNA 直接注入 prompt |
| AI 生图分辨率不够印刷要求 | 手动放大，质量差 | 内置三档输出尺寸（1024px / 2048px / 4096px） |

## 三、功能模块

### 3.1 风格解析（核心功能）

**输入方式**：URL 输入（Playwright 抓取 + Claude 视觉分析）

**输出 —— Style DNA（18 项维度）**：

| 分类 | 项目 |
|------|------|
| 色彩系统 | 主色、辅助色、背景色、强调色、语义色 |
| 字体规范 | 主字体族、标题/正文字重、字号/行高 |
| 空间系统 | 基础间距单位、容器最大宽度 |
| 圆角语言 | 按钮/卡片/输入框圆角值 |
| 组件语言 | 按钮/输入框/卡片/导航风格 |
| 整体风格 | 明暗偏好、信息密度、personality 关键词 |

**解析状态**：进行中（进度条）→ 完成（Style DNA 卡片）→ 失败（错误 + 重试）

---

### 3.2 多模型生图对比（差异化功能）

**流程**：
1. 输入生图描述（自然语言 prompt）
2. 可选：挂载 Style DNA 作为风格约束
3. 选择参与对比的模型（最多 4 个并发）
4. 点击"全部生成" → 结果并排展示

**支持模型（当前实现）**：

| 模型 | ID | API Key | 说明 |
|------|-----|---------|------|
| DALL-E 3 | `gpt-4o` | `openai` | 返回 base64 |
| Gemini Flash Image | `gemini-flash-image` | `google` | 模型：`gemini-2.5-flash-image` |
| Ideogram v2 | `ideogram-v2` | `ideogram` | 服务端转 base64 |
| ComfyUI | `comfyui` | `comfyui_url` + `comfyui_model` | 本地推理，支持高分辨率 |

**API Key 策略**：用户自带（BYOK），存浏览器 localStorage，每次请求随 body 传递

**ComfyUI 输出尺寸选项**：

| 选项 | 尺寸 | 说明 |
|------|------|------|
| 1024px（预览） | 1024×1024 | 快速预览，~3-4 分钟 |
| 2048px Lanczos | 2048×2048 | 方形印刷品，Lanczos 算法 2x 放大 |
| 4096px AI | 4096×4096 | T-shirt / 海报，4x-UltraSharp 神经网络放大 |

**Printify 印刷规格参考**：PNG 格式，透明背景，300 DPI，方形最小 2048×2048，T-shirt 最小 4500×5400px

---

### 3.3 风格库（留存功能）

- 每次解析自动归档
- 卡片展示：来源缩略图、风格色盘、描述摘要
- 支持按 Style DNA 挂载到生图任务

---

### 3.4 设置页

- 各模型 API Key 填写（存 localStorage）
- ComfyUI 服务地址（默认 `http://127.0.0.1:8188`）
- ComfyUI 模型文件名（默认 `flux1-schnell-Q4_K_S.gguf`）

## 四、页面结构（当前实现）

```
/                    首页（URL 输入 → 触发分析）
/library             风格库（所有已分析的 Style DNA 列表）
/library/[id]        单个 Style DNA 详情（API route，供 compare 读取）
/compare             多模型对比生图页
/settings            API Key 设置页
/analyze/[id]        Style DNA 分析结果详情（规划中）
```

## 五、核心用户流程

### 流程 A：风格解析

```
首页输入框 → 粘贴 URL → 点击"分析"
→ 进度展示（Claude 分析中）
→ Style DNA 报告页
→ 自动归档至风格库
→ 引导进入"模型对比"
```

### 流程 B：多模型对比生图

```
进入对比页
→ （可选）挂载 Style DNA
→ 输入生图描述
→ 选择模型 + ComfyUI 输出尺寸
→ 点击"全部生成"
→ 并排查看结果
→ 下载满意的图片（PNG / base64）
```

## 六、非功能性需求

| 项目 | 指标 |
|------|------|
| 风格解析响应时间 | P90 ≤ 15s |
| ComfyUI 生图时间 | 3–5 分钟（本地 M4 16GB） |
| 云端模型生图时间 | 各模型各自计时，超时显示提示 |
| 高分辨率输出 | 4096×4096 AI 放大，满足 T-shirt 印刷需求 |
| 数据安全 | API Key 只存 localStorage，不经服务器持久化 |
